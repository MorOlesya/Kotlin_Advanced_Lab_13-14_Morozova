## Изученные темы

### Generics

**Generics (обобщённые типы)** — написание универсального кода,
работающего с разными типами данных. Когда у свойства может быть разный тип данных, но логика класса остаётся одинаковой, в Kotlin используется generic-тип.

Пример Generics
```kotlin
class Question<T>(
    val questionText: String,
    val answer: T,
    val difficulty: String
)
```

###  enum class

**enum class** применяется, когда:
* существует ограниченный набор возможных значений;
* использование любых других значений недопустимо;
* требуется строгий контроль на уровне компилятора.

Пример использования:
```kotlin
enum class Difficulty {
    EASY,
    MEDIUM,
    HARD
}
```

### data class

**data class** — это класс, предназначенный для хранения данных.

Пример:
```kotlin
data class Question<T>(
    val questionText: String,
    val answer: T,
    val difficulty: Difficulty
)
```

### singleton-объекта (object)

**Singleton и companion object** — объекты, существующие в единственном
экземпляре.

Примеры использования:

* статистика игрока в мобильной игре;
* доступкаппаратному устройству (динамик, камера);
* работа судалённым источником данных;
* система аутентификации (один активный пользователь).

Пример кода:
```kotlin
object StudentProgress {
    var total: Int = 10
    var answered: Int = 3
}
```

### Companion object

В Kotlin объекты можно объявлять внутри классов. Если такой объект помечен как companion, его свойства доступны напрямую через имя класса.

Пример кода:
```kotlin
companion object StudentProgress {
    var total: Int = 10
    var answered: Int = 3
}
```

###  extension-свойства и extension-функции

**Extension-функции и свойства** — расширение возможностей существующих
классов без наследования.

Extensions позволяют:

* добавлять новые свойства и функции к уже существующим типам;
* использовать их через точечную нотацию (.), как будто они были частью класса;
* не изменять исходный код класса.

Важно:

* расширения не изменяют сам класс,
* они лишь добавляют удобный синтаксис поверх него.

Пример кода extension-свойства:
```kotlin
val Quiz.StudentProgress.progressText: String
    get() = "${answered} of ${total} answered"
```

Пример кода extension-функции:
```kotlin
fun Quiz.StudentProgress.printProgressBar() {
    repeat(Quiz.answered) { print("▓") }
    repeat(Quiz.total - Quiz.answered) { print("▒") }
    println()
    println(Quiz.progressText)
}
```

### Scope-functions

**Scope-функции** — удобные функции для работы с объектами и их контекстом. **Scope-функции** позволяют работать со свойствами и методами объекта, не повторяя имя переменной, если это избыточно.

**Scope-функции** — это функции высшего порядка, которые:

* принимают лямбда-выражение;
* временно меняют контекст (scope) внутри этого блока;
* позволяют обращаться к объекту через it или this.
